# 流程图（Flowchart）

## 这是什么图

展示业务流程、决策路径和操作步骤的图表。每个步骤用特定形状表示（开始/结束、处理、判断、输入/输出），步骤之间用箭头连接。适合描述业务逻辑、算法流程、审批流程、用户操作路径。

## 适用场景

- 业务审批流程设计
- 用户操作路径分析
- 算法逻辑伪代码可视化
- 异常处理与重试流程
- 订单状态流转
- 客服工单处理流程

## 你怎么描述

用自然语言回答以下问题（不需要全答，AI 会追问你漏掉的部分）：

1. **流程从哪里开始？到哪里结束？**
   比如："从用户提交订单开始"到"订单创建成功或失败"结束。
   每个流程图必须有明确的起点和终点。

2. **流程有哪些步骤？按顺序描述。**
   例如：
   1. 用户提交订单
   2. 系统验证用户登录状态
   3. 系统验证商品库存
   4. 如果库存充足 → 扣减库存 → 创建订单 → 返回成功
   5. 如果库存不足 → 返回"库存不足"提示 → 结束

   每个步骤用简短动词短语描述（"验证库存"、"扣减库存"、"创建订单"）。

3. **有哪些判断（分支）？每个分支的条件和走向是什么？**
   像这样描述：
   - **判断：库存是否充足？**
     - 是 → 继续下单流程
     - 否 → 提示用户"库存不足"→ 结束
   - **判断：用户是否登录？**
     - 是 → 进入下单页
     - 否 → 跳转登录页
   每个判断必须有"是"和"否"两个分支。

4. **有没有特殊要求？**
   比如：
   - "正常流程用蓝色，异常/失败流程用红色"
   - "标注每个步骤的负责人/系统（如：人工审核、系统自动）"
   - "关键步骤用加粗或大号字体"
   - "超时场景单独用虚线标注"

## 示例

### 示例1：用户登录流程

> 画一个用户登录的流程图。
>
> 开始 → 用户输入账号密码 → 系统验证格式 →
> 判断：格式是否正确？
> - 是 → 调用登录 API → 判断：账号密码是否匹配？
>   - 是 → 生成 JWT Token → 跳转主页 → 结束
>   - 否 → 返回"账号或密码错误"→ 回到输入账号密码
> - 否 → 返回"请输入正确的邮箱格式"→ 回到输入账号密码
>
> 异常流程用红色虚线标注。

### 示例2：电商订单处理流程

> 画一个电商订单处理的完整流程图。
>
> 开始（用户提交订单）→
> 判断：用户是否已登录？
> - 否 → 跳转登录页 → 用户登录 → 判断：登录是否成功？
>   - 否 → 提示"登录失败"→ 结束
>   - 是 → 继续
> - 是 → 继续
>
> 系统验证商品库存 →
> 判断：库存是否充足？
> - 否 → 提示"库存不足，请调整数量"→ 结束
> - 是 → 锁定库存（预占）→ 创建订单（状态=待支付）→
>
> 判断：用户是否在 30 分钟内完成支付？
> - 是 → 更新订单状态=已支付 → 扣减库存（确认）→ 通知仓库发货 →
>   更新订单状态=已发货 → 判断：用户是否在 7 天内确认收货？
>   - 是 → 更新订单状态=已完成 → 结束
>   - 否 → 系统自动确认收货 → 更新订单状态=已完成 → 结束
> - 否 → 取消订单 → 释放预占库存 → 更新订单状态=已取消 → 结束
>
> 正常流程用蓝色，超时/取消流程用红色标注，支付超时用虚线。

## 约束（AI 生成时必须遵守）

- **布局方向**：主流程从上到下，分支在判断节点处左右展开。
- **形状规范**：
  - 开始/结束：椭圆（ellipse），绿色（fillColor=#d5e8d4）
  - 处理步骤：圆角矩形（rounded=1），蓝色（fillColor=#dae8fc）
  - 判断/分支：菱形（rhombus），黄色（fillColor=#fff2cc）
  - 输入/输出：平行四边形（shape=parallelogram），橙色（fillColor=#ffe6cc）
  - 异常/错误：红色（fillColor=#f8cecc）
- **步骤间距**：垂直间距 ≥ 80px，判断分支水平间距 100–120px。
- **判断分支**：每个菱形判断节点必须有两条出边（Yes 和 No），边的 `value` 属性标注"是"/"否"或具体条件。
- **边标签背景**：所有带文字标签的边，添加 labelBackgroundColor=#FFFFFF。
- **路径区分**：主路径（正常流程）用实线；异常/回退/超时路径用虚线（dashed=1;dashPattern=8 4;）。
- **起点终点**：有且仅有一个开始节点（顶部中央）；至少一个结束节点（底部）。
- **边交叉**：通过调整分支顺序或添加中间节点来避免边线交叉。
- **坐标对齐**：所有坐标必须是 10px 的整数倍。
