# C4 Model

## What This Is

A diagram showing software architecture at four zoom levels: System Context (C1), Container (C2), Component (C3), and Code (C4). Best for system documentation, architecture decision records, and team onboarding.

## When to Use

- System architecture documentation (ADRs)
- Onboarding new team members
- Architecture review and decision-making
- Zooming from macro to micro views of a system

## How to Describe Your System

1. **Which C4 level do you need?**
   - C1 (System Context): System + external users/systems
   - C2 (Container): Apps, databases, file systems inside the system
   - C3 (Component): Components within each container
   - C4 (Code): Class level (typically use UML class diagrams instead)

2. **C1: What's the system called? Who are the users and external systems?**
   e.g., "E-Commerce Platform" (core system), users: "Customer" and "Admin", external: "Stripe" and "Shipping Provider"

3. **C2: What containers are inside the system?**
   e.g., Web App (React), API Service (Spring Boot), MySQL Database, Redis Cache

4. **C3: What components are inside each container?**
   e.g., Inside API Service: "Order Controller", "Payment Service", "Inventory Service", "Notification Service"

## Constraints (AI must follow)

- **布局方向**：从上到下分层排列。C1：Person 在上 → System 在中 → External 在下。C2/C3：系统边界内从左到右排列容器/组件。层间垂直间距 ≥ 120px，同层水平间距 100–120px。
- **形状规范**：
  - Person/用户：actor（shape=actor;whiteSpace=wrap;html=1;）
  - 软件系统/容器/组件：圆角矩形（rounded=1;whiteSpace=wrap;html=1;）
  - 外部系统：圆角矩形 + 虚线边框（dashed=1;dashPattern=8 4;）
  - 数据库/数据存储：圆柱体（shape=cylinder3;whiteSpace=wrap;html=1;boundedLbl=1;backgroundOutline=1;size=15;）
  - 系统边界/容器边界：虚线框（rounded=1;dashed=1;dashPattern=8 4;fillColor=none;）
- **颜色语义**：不同层级用不同语义角色，色值由所选风格 JSON 查表确定——
  - C1：系统 → role: service，外部系统 → role: external
  - C2：容器 → role: service，数据库 → role: database（圆柱体）
  - C3：组件 → role: gateway
  - C4：类/模块 → role: external
  - **查表方式**：读所选风格 JSON → 查 `roles` 字段找 palette 槽位 → 查 `palette` 字段取 fillColor/strokeColor
- **数量约束**：C2 ≤ 4 个容器，C3 ≤ 6 个组件。
- **边规范**：边使用正交路由（edgeStyle=orthogonalEdgeStyle;rounded=1;orthogonalLoop=1;），箭头使用 blockThin（endArrow=blockThin;endFill=1;）。外部交互用虚线（dashed=1;dashPattern=8 4;）。边标签格式：`"描述 [技术]"`，所有边标签添加 labelBackgroundColor=#FFFFFF。
- **坐标对齐**：所有坐标必须是 10px 的整数倍。
- **标题**：图上方添加标题注明 C1/C2/C3 级别，fontSize=14, fontStyle=1。
