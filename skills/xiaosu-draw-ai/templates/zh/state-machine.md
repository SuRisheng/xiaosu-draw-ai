# 状态机图（State Machine Diagram）

## 这是什么图

展示对象或系统的状态集合、状态之间的转换以及触发转换的事件。适合生命周期建模、协议设计、工作流状态管理。

## 适用场景

- 订单生命周期（待支付→已支付→已发货→已完成→已取消）
- 用户账号状态（正常→冻结→注销）
- 工单流转（新建→处理中→已解决→已关闭）
- 协议状态机（TCP 连接、OAuth 流程）
- 审批流程状态

## 你怎么描述

1. **这个状态机描述什么对象/系统？**
   比如："订单的生命周期"、"用户认证流程"、"TCP 连接状态"

2. **有哪些状态？**
   比如：草稿、待审核、已发布、已归档、已删除
   每个状态用一个短语描述（简短，建议 2-6 个状态）

3. **状态之间怎么转换？触发条件是什么？**
   比如：
   - 草稿 → 待审核：用户点击"提交审核"
   - 待审核 → 已发布：管理员点击"审核通过"
   - 待审核 → 草稿：管理员点击"驳回"
   - 已发布 → 已归档：系统自动（发布后 90 天）

4. **有没有特殊标注？**
   比如："用红色标注错误状态"、"标注守卫条件（如 [余额>0]）"

## 示例

### 示例1：订单状态机

> 画一个电商订单的状态机图。
> 状态：待支付 → 已支付 → 已发货 → 已完成 → 已取消
> 待支付 → 已支付：用户完成支付
> 待支付 → 已取消：用户取消 或 30分钟超时
> 已支付 → 已发货：仓库发货
> 已发货 → 已完成：用户确认收货 或 7天自动确认

### 示例2：用户认证状态机

> 画一个用户登录认证的状态机图。
> 初始 → 未登录
> 未登录 → 登录中：用户输入凭证
> 登录中 → 已登录（主页）：验证成功
> 登录中 → 未登录：验证失败 [retryCount < 3]
> 登录中 → 已锁定：验证失败 [retryCount >= 3]
> 已登录 → 未登录：用户登出 或 token 过期
> 已锁定 → 未登录：管理员解锁 或 30分钟自动解锁

## 约束（AI 生成时必须遵守）

- **布局方向**：初始状态在最左（或最上），终态在最右（或最下）。简单状态链从左到右排列，复杂图以平衡布局展开。状态之间水平间距 100px，垂直间距 120px。页面边距 ≥ 40px。
- **形状规范**：
  - 初始状态：实心小圆（ellipse;whiteSpace=wrap;html=1;fillColor=#6c8ebf;strokeColor=#6c8ebf;），直径 20–30px
  - 终态：双层圆（ellipse;whiteSpace=wrap;html=1;fillColor=#FFFFFF;strokeColor=#000000;strokeWidth=2;）
  - 普通状态：圆角矩形（rounded=1;whiteSpace=wrap;html=1;）
  - 选择伪状态（choice）：菱形（rhombus;whiteSpace=wrap;html=1;）
  - 复合状态（含子状态）：swimlane（swimlane;startSize=30;）
- **颜色语义**：
  - 普通状态：蓝色（fillColor=#dae8fc, strokeColor=#6c8ebf）
  - 终态：绿色（fillColor=#d5e8d4, strokeColor=#82b366）
  - 错误/异常状态：红色（fillColor=#f8cecc, strokeColor=#b85450）
  - 初始状态：深蓝（fillColor=#6c8ebf, strokeColor=#6c8ebf）
- **边规范**：边使用正交路由（edgeStyle=orthogonalEdgeStyle;rounded=1;orthogonalLoop=1;jettySize=auto;html=1;），箭头用 block（endArrow=block;endFill=1;）。转换标签格式：`事件 [守卫条件] / 动作`。自转换（同一状态到自身）使用弧形边（orthogonalLoop=1;jettySize=auto;），放在状态上方或右侧。
- **数量约束**：有且仅有一个初始状态。可有一个或多个终态。choice 节点的每条出边必须标注条件（如 `[是]`/`[否]` 或具体守卫）。
- **坐标对齐**：所有坐标必须是 10px 的整数倍。
